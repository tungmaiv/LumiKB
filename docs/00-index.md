## Technical Stack

### Backend
- **Framework:** FastAPI + Pydantic v2
- **ORM:** SQLAlchemy 2.0 async
- **Auth:** FastAPI-Users with JWT cookies
- **Database:** PostgreSQL 16
- **Cache:** Redis 7
- **Migrations:** Alembic
- **Task Queue:** Celery + Celery Beat

### Frontend
- **Framework:** Next.js 15 (App Router)
- **UI Library:** shadcn/ui + Tailwind CSS
- **State:** Zustand + React Query (TanStack Query)
- **Forms:** React Hook Form + Zod

### Infrastructure
- **Container:** Docker Compose
- **Object Storage:** MinIO
- **Vector DB:** Qdrant
- **Graph DB:** Neo4j 5.x (GraphRAG integration)
- **LLM Proxy:** LiteLLM (routes to local Ollama or cloud providers)

### LLM Models (Default: Local Ollama)
- **Generation:** Ollama gemma3:4b (offline-capable)
- **Embedding:** Ollama nomic-embed-text (768 dimensions)
- **Cloud Options:** OpenAI, Anthropic, Google Gemini (via LiteLLM)

### Testing
- **Backend:** pytest, pytest-asyncio, testcontainers
- **Frontend:** Vitest, React Testing Library
- **E2E:** Playwright

## Key Decisions Summary

| Decision | Source | Details |
|----------|--------|---------|
| Architecture | ADR-001 | Python + FastAPI monolith |
| Vector DB | ADR-002 | Qdrant with collection-per-KB isolation |
| Consistency | ADR-003 | Transactional outbox pattern |
| Audit Logging | ADR-004 | PostgreSQL audit table (INSERT-only) |
| Core Differentiator | ADR-005 | Citation-first architecture |
| GraphRAG | ADR-006 | Neo4j for entity graphs, domain-aware extraction, vector-first retrieval |
| Frontend | All Docs | Next.js 15 + shadcn/ui + Zustand |
| Storage | All Docs | MinIO (blob) + PostgreSQL (metadata) + Neo4j (graph) |
| MVP Approach | Brainstorming | 8 epics: Foundation → KB → Search → Chat → Admin → Lifecycle → DevOps → GraphRAG |
